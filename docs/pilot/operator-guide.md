# Operator Guide

Day-to-day dashboard usage for support and QA teams.

## Login

1. Go to `/dashboard`
2. Register or login with email/password
3. Token stored in browser — click **Logout** when done

## Daily workflow

| Task | Section |
|------|---------|
| Monitor session volume | Overview |
| Investigate failed call | Sessions → Replay |
| Review latency spikes | Latency |
| Check quality regressions | Evaluations, Alerts |
| Handle escalations | Handoffs |
| Update knowledge | Knowledge Base |

## Replay workflow

1. **Sessions** → select session ID
2. **Replay** → review messages, tool calls, evaluations
3. Share signed replay link with engineering if needed

## Handoff workflow

1. **Handoffs** → view pending queue
2. Open context panel for transcript summary
3. Resolve or assign to human agent
4. Close ticket in external system if integrated

## Alerts

Regression alerts fire on threshold breaches (quality, latency, escalations). Review **Alerts** section weekly.
